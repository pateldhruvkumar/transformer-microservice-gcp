## Project overview

EAI6010 Assignment 5 — A PyTorch Transformer language model (trained on Tiny Shakespeare) wrapped in a FastAPI microservice and deployed to Google Cloud Run.

- `notebooks/` — Assignment 4 Jupyter notebook (training reference; not part of the service)
- `shakespeare-api/` — The deployable microservice (all active development happens here)

## Development commands

All commands run from inside `shakespeare-api/`.

```bash
# One-time setup
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt

# Run locally (hot-reload)
uvicorn app.main:app --reload --port 8080

# Re-train model and regenerate artifacts into artifacts/
python scripts/export_artifacts.py

# Deploy to Cloud Run
gcloud run deploy shakespeare-api \
  --source . --region us-central1 \
  --allow-unauthenticated --memory 1Gi --timeout 60

# Stream logs from the live service
gcloud run logs read shakespeare-api --region us-central1
```

## Architecture

### Package layout

```
shakespeare-api/
├── app/
│   ├── main.py     # FastAPI factory, lifespan (artifact loading), CORS
│   ├── routes.py   # Route handlers: GET /, GET /health, POST /generate
│   ├── schemas.py  # Pydantic: GenerateRequest, GenerateResponse
│   ├── model.py    # ML classes (TransformerModel, Vocab, generate())
│   └── state.py    # AppState singleton — shared loaded model + vocab
├── artifacts/      # gitignored — best_model.pt, vocab.json, hyperparams.json
└── scripts/
    └── export_artifacts.py  # Full retrain → writes to artifacts/
```

### Request flow

```
POST /generate
  → routes.py validates request (Pydantic)
  → calls model.generate(state.model, state.vocab, prompt, ...)
  → model.py tokenizes with basic_english_tokenize()
  → autoregressive loop: TransformerModel.forward() → softmax → top-k sample or argmax
  → returns space-joined token string
```

### Startup (lifespan in `main.py`)

`main.py` loads the three artifacts from `artifacts/` once at startup in this order:
1. `vocab.json` → `Vocab.from_json()` → `state.vocab`
2. `hyperparams.json` (falls back to hardcoded defaults if missing)
3. `best_model.pt` → `TransformerModel.load_state_dict()` → `state.model`

`state.py` holds the module-level `AppState` singleton imported by both `main.py` and `routes.py` — this avoids circular imports.

### Critical constraint — model class fidelity

The classes in `app/model.py` **must exactly match** the notebook. In particular, `TransformerEncoderLayer` is constructed without `batch_first=True` (triggers a harmless PyTorch UserWarning). Changing this will break weight loading from `best_model.pt`.

Hyperparameters: `emsize=200`, `d_hid=200`, `nlayers=2`, `nhead=2`, `dropout=0.3`, vocab ≈ 10,926 tokens.

### Artifacts

Generated by `scripts/export_artifacts.py` (full retrain, ~2.5 min on Apple Silicon MPS). Weights are saved to CPU so they load cleanly on Cloud Run without CUDA/MPS.

### Deployment

Cloud Run uses CPU-only inference. `requirements.txt` pins the `+cpu` torch wheel. The Dockerfile entry point is `uvicorn app.main:app`, with `$PORT` wired via the Cloud Run environment variable.
